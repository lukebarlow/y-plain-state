This is a second rewrite of of the Yjs jigsaw example using the y-plain-state proxy objects, this time re-implementing the UI in React.

Although this might not seem like a simplification of the original example, since there are more files involved, the way the functionality is divided up means that this might be a good pattern for larger more complex applications. Each react component is almost entirely self contained, and makes its own modifications to the state when it changes (which are synchronised automatically).

Besides the use of React, the other main change is that every pixel of movement is transmitted to other clients. This makes it more responsive, but leads to slightly jerky movement on the receiving client.

Again, you can inspect and modify the state directly with commands such as 
JSON.stringify(state, null, 2)
and 
state.piece2.translation.x += 40
